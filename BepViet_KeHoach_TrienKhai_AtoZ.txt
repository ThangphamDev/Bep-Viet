
BẾP VIỆT – KẾ HOẠCH TRIỂN KHAI TỪ A→Z (BACKEND NODE.JS + FLUTTER DART)
==============================================================================
MỤC TIÊU
- Ứng dụng giải quyết “Hôm nay ăn gì?” với gợi ý theo vùng miền, mùa vụ, ngân sách.
- Lập kế hoạch tuần, tạo danh sách đi chợ, quản lý “Tủ lạnh của tôi”.
- Cộng đồng công thức, gói Premium với hồ sơ gia đình và cảnh báo dinh dưỡng.
- Backend Node.js (khuyến nghị NestJS + TypeScript). Mobile FE Flutter (Dart). DB MySQL.

LỘ TRÌNH & TASKS (ROADMAP)
------------------------------------------------------------------------------
PHASE 0 – Khởi tạo & CI/CD (1 tuần)
- [DevOps] Quy ước repo, branch, commit (Conventional Commits), code style (ESLint, Prettier).
- [Infra] Docker Compose: mysql, adminer, api, flutter web (dev).
- [CI] GitHub Actions: Lint + Test + Migrate DB + Build Docker.
- [Security] .env mẫu & vault, git-secrets, renovate/bot.
Deliverables: /docker, /infra, pipeline CI; README khởi tạo dự án.

PHASE 1 – Nền tảng dữ liệu & Auth (2 tuần)
- [DB] Áp dụng schema MySQL đã cung cấp (22+ bảng core + mở rộng).
- [Auth] Đăng ký/Đăng nhập (email + password), JWT, refresh token, RBAC (user, admin).
- [User Profile] Thêm region/subregion, khẩu vị, hạn chế dinh dưỡng.
Deliverables: Migrations, Seed cơ bản; /auth, /users module; Postman collection.

PHASE 2 – Công thức & Gợi ý theo vùng/mùa/ngân sách (3 tuần)
- [Recipes] CRUD recipes, variants theo Bắc/Trung/Nam; bước nấu, ảnh.
- [Seasonality] Seasons + ingredient_seasonality; ưu tiên gợi ý theo mùa.
- [Budget] ingredient_prices theo vùng; ước tính chi phí món và bữa.
- [Suggest API] Gợi ý theo (region, season, servings, budget).
Deliverables: /recipes, /variants, /ingredients, /prices, /suggestions.

PHASE 3 – Kế hoạch tuần + Danh sách đi chợ + Tủ lạnh (3 tuần)
- [Planner] Tạo meal_plan (7 ngày), tránh lặp, cân bằng chi phí.
- [Shopping] Tổng hợp nguyên liệu từ kế hoạch → shopping_list + group theo quầy.
- [Pantry] Quản lý tồn kho, cảnh báo HSD, trừ tồn khi nấu.
Deliverables: /meal-plans, /shopping-lists, /pantry.

PHASE 4 – Premium (Family Profile) & Cảnh báo (2 tuần)
- [Premium] Subscriptions, family_profiles/members, allergies, hạn chế.
- [Advisory] Cảnh báo cay/dị ứng/huyết áp/tiểu đường theo hồ sơ.
Deliverables: /subscriptions, /family, /advisory endpoints.

PHASE 5 – Cộng đồng & Moderation (2 tuần)
- [Community] Đăng công thức, bình luận, đánh giá, like/favorite.
- [Admin] Duyệt, gắn Featured, phân tích cơ bản.
Deliverables: /community, /comments, /ratings, /moderation, /analytics.

PHASE 6 – Polish & Growth (liên tục)
- i18n, Performance, Offline, Push Notification, A/B test, Deep links.
- Tích hợp siêu thị (giá theo chợ/siêu thị, giao hàng), Video hướng dẫn.

API DESIGN (OpenAPI tóm tắt)
------------------------------------------------------------------------------
Auth
- POST /auth/register {email, password, name, region, subregion}
- POST /auth/login {email, password} -> {accessToken, refreshToken}
- POST /auth/refresh {refreshToken}
- POST /auth/logout

Users
- GET  /users/me
- PATCH /users/me {name, region, subregion, preferences[], favorite_cuisines[]}
- GET  /users/:id (admin)

Regions & Seasonality
- GET /regions (Bắc/Trung/Nam + subregions)
- GET /seasons (Xuân/Hạ/Thu/Đông)
- GET /ingredients/seasonality?ingredient_id=...

Ingredients & Prices
- GET /ingredients?search=...&category=...
- POST/PUT/DELETE /ingredients (admin)
- GET /ingredient-prices?ingredient_id=&region=&date=
- POST/PUT/DELETE /ingredient-prices (admin)

Recipes
- GET /recipes?region=&budget=&difficulty=&tag=
- POST /recipes (admin/author)
- GET /recipes/:id
- PATCH /recipes/:id
- DELETE /recipes/:id (admin)
- GET /recipes/:id/variants
- POST /recipes/:id/variants (admin/author)
- GET /recipes/:id/steps /:ingredients
- POST /recipes/:id/media (ảnh)

Suggestions (core)
- POST /suggestions/search
  body: { region, season, servings, budget, spice_pref, pantry_ids[], exclude_allergens[], max_time }
  resp: [{ recipe_id, variant_region, total_cost, items:[{ingredient_id, qty, est_cost}], season_score, reason }]

Meal Plans
- POST /meal-plans/generate
  body: { week_start, region, budget_per_meal, servings, constraints:{no_repeat: true, nutrition_balance:true} }
- GET /meal-plans/:id
- PATCH /meal-plans/:id (điều chỉnh thủ công)
- POST /meal-plans/:id/add-recipe {day, meal_type, recipe_id, variant_region}
- DELETE /meal-plans/:id/remove-recipe {day, meal_type}

Shopping Lists
- POST /shopping-lists/from-meal-plan {meal_plan_id, include_pantry:true}
- GET /shopping-lists/:id
- PATCH /shopping-lists/:id/check-item {item_id, checked:true}
- GET /shopping-lists/:id/export?format=pdf|csv

Pantry
- GET /pantry
- POST /pantry {ingredient_id, quantity, unit, expire_date, location}
- PATCH /pantry/:id
- DELETE /pantry/:id
- POST /pantry/consume {ingredient_id, quantity}

Premium – Subscriptions & Family
- GET /subscriptions/my
- POST /subscriptions/checkout (stub) -> {status:active}
- GET /family
- POST /family {name, note}
- POST /family/members {name, age, allergies[], spice_level, diet_flags[]}
- PATCH /family/members/:id
- Advisories
  - POST /advisory/check
    body: { recipe_id, variant_region, family_member_ids[] }
    resp: [{member_id, issues:[{type:ALLERGY|SPICE|SODIUM, level:WARN|BLOCK, message}]}]

Community & Moderation
- GET /community/recipes?region=&tag=&status=Approved
- POST /community/recipes (user)
- GET /community/recipes/:id
- POST /community/recipes/:id/comments
- POST /community/recipes/:id/ratings
- POST /community/recipes/:id/favorite
- Admin moderation
  - POST /moderation/recipes/:id/approve|reject|feature
- Analytics (admin)
  - GET /analytics/recipes/top?range=7d

Error model (chuẩn hoá)
- { error: { code: "VALIDATION_ERROR", message: "...", details: [] } }

LOGIC GỢI Ý (Pseudo)
- Input: region, season, servings, budget, pantry, spice_pref, max_time
- For each recipe -> chọn variant theo region -> ước tính cost theo ingredient_prices(region)
- season_score = avg(availability% của nguyên liệu theo season)
- base_score = season_score + (base_region==region ? +10 : 0)
- budget_bonus = (total_cost <= budget ? +15 : - (total_cost - budget)/5000 )
- pantry_bonus = -cost của nguyên liệu đã có sẵn (nếu trừ tồn)
- time_penalty = (timeMin > max_time ? -20 : 0)
- sort desc by final_score

STATE MANAGEMENT
- Có thể dùng Bloc/Cubit hoặc Riverpod. Gợi ý: Cubit cho từng feature (suggest_cubit, planner_cubit).

API CLIENT (Dart)
- api_client.dart: Dio/Http với baseURL, interceptors (JWT), retry, error mapping.
- Model codegen: json_serializable.

TESTING
- Backend: Jest unit + e2e (Supertest). Test scoring suggestions (đầu vào cố định, kiểm tra top-1).
- Mobile: Widget tests cho mỗi page; golden tests cho card.
- Postman/Bruno collection: auth, recipes, suggestions, planner, shopping, pantry, community, premium.

VERSIONING & MIGRATIONS
- DB migration (Prisma/TypeORM). Tách seed (regions, seasons, units, store_sections).
- OpenAPI version: /v1, chuẩn hoá error model.

SECURITY & PERFORMANCE
- Rate limiting, helmet, CORS, input validation (class-validator).
- Caching: ingredient_prices theo region, suggestions cache theo (region, season, budget bucket).
- Pagination & projections cho lists, N+1 tránh bằng JOIN/select.

OBSERVABILITY
- Logger (pino/winston), request-id, audit trail (moderation, admin).
- Metrics: Prometheus endpoint, tracing (OpenTelemetry).
